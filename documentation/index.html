trashsync - Waste Collection Management System (Documentation)

1. Introduction

trashsync is a comprehensive web application built with modern technologies to streamline waste collection, processing, and management operations. This system combines the power of Express.js for robust backend API development and React.js for an intuitive, responsive frontend experience.

Technology Stack

Backend:

  • Express.js - Fast, unopinionated web framework for Node.js with REST API architecture

Frontend:

  • React.js with modern JavaScript (ES6+)

Database:

  • MySQL

Authentication:

  • JWT (JSON Web Tokens) - Provides stateless, secure authentication for API endpoints

Styling:

  • Bootstrap with responsive design principles

Local Installation (Windows)

Follow these steps to run trashsync locally on your computer.

1) Prerequisites
  • Node.js 20.x (recommended) — https://nodejs.org/en
  • Xampp — https://www.apachefriends.org/
  • Git (optional) — https://git-scm.com/downloads
2) Database Setup (XAMPP)

Use XAMPP's MySQL (MariaDB) and phpMyAdmin for local database.

  1. Install & open XAMPP Control Panel.
  2. Start Apache and MySQL services.
  3. Open phpMyAdmin: http://localhost/phpmyadmin.
  4. Create database: trashsync (Collation: utf8mb4_general_ci).
  5. Go to Import tab and choose database.sql from the project root.
  6. Click Import and wait until success confirmation appears.
3) Backend Setup
  1. Open terminal in backend folder.
  2. Install dependencies: 
    npm install
  3. Copy env.example to .env and update values (XAMPP defaults): 
    NODE_ENV=development
PORT=3000
BACKEND_URL=http://localhost:3000
FRONTEND_URL=http://localhost:5173
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=
DB_NAME=trashsync
JWT_SECRET=change_me_securely
JWT_EXPIRES_IN=24h
  4. (Optional) Run migrations if needed: 
    npm run db:migrate
    npm run db:seed
  5. Start API server: 
    npm run dev

    Server: http://localhost:3000

4) Frontend Setup
  1. Open terminal in frontend folder.
  2. Open .env and update value 
    VITE_API_BASE_URL=http://localhost:3000
  3. Install dependencies: 
    npm install
  4. Run development server: 
    npm run dev

    App: http://localhost:5173

5) Test Login
  • Open http://localhost:5173/login
  • Use the credentials from the Reviewer Credentials section or your own created user.
Troubleshooting (Local)
  • PORT in use: change PORT in .env or stop existing process.
  • CORS error: ensure FRONTEND_URL matches http://localhost:5173.
  • Database error: ensure XAMPP MySQL is running and DB_PASSWORD is empty if you didn't set one.
  • phpMyAdmin not loading: verify Apache is started in XAMPP.

2. System Requirements

Before installing and running trashsync, ensure your server meets the following requirements:

1. Web Server

  • Apache (with mod_rewrite enabled), Nginx, or any HTTP server capable of reverse proxy

2. Node.js Runtime:

  • Node.js >= 20.x
  • NPM >= 9.x or Yarn >= 1.22.x

3. Database:

  • MySQL >= 8.0
Required Node.js Dependencies

The backend application requires the following core dependencies:

  • Express: Web application framework
  • Drizzle-orm: JavaScript ORM for database operations
  • Mysql2: MySQL client for Node.js
  • Jsonwebtoken: JWT implementation for authentication
  • Bcryptjs: Password hashing utility
  • Cors: Cross-Origin Resource Sharing middleware
  • Dotenv: Environment variable management
  • Zod: Input validation middleware
Recommended Environment Configuration

For optimal performance and security of the trashsync Express.js application, configure the following environment variables:

  • NODE_ENV=production (for production deployment)
  • PORT=3000(or your preferred port)
  • post_max_size 64M
  • DB_HOST=localhost
  • DB_PORT=3306
  • DB_USER=your_db_user
  • DB_PASSWORD=your_db_password
  • DB_NAME=trashsync
  • JWT_SECRET=your_secure_jwt_secret
  • JWT_EXPIRES_IN=24h

Installation Guide: Frontend

1. Extract frontend.zip:

Unzip the frontend.zip then go to frontend folder and open the terminal in that folder.

2. Set API Endpoint:

- Create/Edit frontend/.env file.

- Set the API base URL to point to your backend, for example:

VITE_API_BASE_URL=https://api.your-domain.com

Ensure the URL matches the deployed backend (e.g., https://api.your-domain.com or http://localhost:3000 for local testing).

3. Build file:

1. Open your source-code/frontend folder and open terminal or command prompt

2. Type npm run build to build the frontend file with upper information

3. It will take some time and it will build a file in the dist folder.

4. Zip and Upload the File (Local Build):

1. Zip the dist folder with any compressor (e.g., WinRAR, 7-Zip).

5. Click Upload, select the zipped dist folder, and extract it to the target directory.

Ensure the dist folder contains index.html, static/, and other assets.

6. Now Extract the file:

1. Select the file you wish to uncompress

2. Hit Extract icon

3. Or right-click on the file and use Extract option

4. Define the path you wish to extract the file to

5. Once done, hit Extract File(s)


7. Create .htaccess file (for React Router)

Inside the folder where index.html is located, create a .htaccess file with the following content:

Options -MultiViews
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.html [QSA,L]

# php -- BEGIN cPanel-generated handler, do not edit
# Set the “ea-php83” package as the default “PHP” programming language.

  AddHandler application/x-httpd-ea-php83___lsphp .php .php8 .phtml

# php -- END cPanel-generated handler, do not edit

Final Checklist:
  • .env file has correct API URL
  • npm run build completed without errors
  • dist folder zipped and uploaded
  • All assets (HTML, static files) are present after extraction
  • .htaccess file created and placed correctly

Authentication

trashsync API uses JWT for authentication. All protected routes require a valid token.

POST /login

Authenticate user and receive access token

Request Body:
{
  "email": "user@example.com",
  "password": "password123"
}
Success Response:
{
  "success": true,
  "message": "Login successful",
  "data": {
    "user": {
      "id": 1,
      "name": "John Doe",
      "email": "user@example.com"
    },
    "token": "1|xyz123abc456def..."
  }
}
POST /forgot-password

Request password reset

POST /reset-password

Reset password with token

POST /logout

Logout and revoke current token

Troubleshooting

Common issues and their solutions:

1. Database Connection Issues:
  • Verify database credentials in .env file
  • Ensure MySQL server is running
  • Check database name and user permissions
  • Verify SSL settings for production
2. JWT Authentication Errors:
  • Ensure JWT_SECRET is set in environment variables
  • Check token expiration time
  • Verify token format in Authorization header
3. File Upload Issues:
  • Check file size limits (5MB default)
  • Verify allowed file types
  • Ensure uploads directory has write permissions
4. CORS Issues:
  • Update FRONTEND_URL in environment variables
  • Check CORS configuration in app.js
5. Node.js Application Won't Start:
  • Check all required environment variables are set
  • Verify Node.js version compatibility
  • Run npm install to ensure all dependencies are installed
  • Check application logs for specific error messages

FAQs

How do I change the API base URL?

Edit frontend/.env and set VITE_API_BASE_URL to your backend URL. Restart the dev server after changes.

Database import fails in phpMyAdmin

Ensure XAMPP MySQL is running, select the correct database, and import database.sql. For large files, increase upload_max_filesize and post_max_size in php.ini.

Frontend shows CORS error

Update FRONTEND_URL in backend .env to match your frontend origin (e.g., http://localhost:5173) and restart the backend.

How do I change favicon/title?

Default favicon is in frontend/public/images/fav.png. Title and runtime favicon come from settings via the app; for local, ensure VITE_API_BASE_URL is correct.
